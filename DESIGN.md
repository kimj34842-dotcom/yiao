# DESIGN.md

> "专业扎实但有温度，活泼开朗且治愈。像阳光下的吉伊乐园，明亮通透，马卡龙多色点缀，圆润温暖，充满故事感。"

## 1. 视觉主题与气质 (Visual Theme & Atmosphere)

- **风格定位**：吉伊乐园风治愈极简 (Chiikawa Land Healing Minimal)
- **气质关键词**：专业扎实、活泼温暖、阳光明亮、故事感
- **核心原则**：
  - **多色点缀**：拒绝单一粉紫，引入经典的 Chiikawa 角色马卡龙四色（吉伊粉、小八蓝、乌萨奇黄、飞鼠紫）进行模块与标签点缀。
  - **层次分明**：拒绝白色卡片贴白色背景，通过“纯白卡片 + 2px 暖褐边框 + 4px 实体彩色硬投影”的设计，让卡片在米白背景上跃然而出。
  - **克制精致**：不堆砌幼稚的装饰，通过精致的排版、多色 Tag 系统和手绘风微交互提升专业度。
  - **轻盈动效**：参考 Chiikawa 角色的漂浮感，采用轻盈的 Spring 弹性动效，悬停时卡片上浮且投影加深。

## 2. 颜色系统 (Color Token System)

所有颜色必须通过 Tailwind 配置文件或 CSS 变量引用，严禁在组件中硬编码 Hex 值。

```css
:root {
  /* 基础背景与容器 */
  --bg: #FAF8F5;                          /* 页面背景：温暖米奶油色 */
  --surface: #FFFFFF;                     /* 卡片/容器底色：纯白轻盈 */
  --surface-alt: #F5EFE6;                 /* 次级背景：交替 Section 背景 */

  /* 边框与线条 */
  --border: #EFEAE2;                      /* 默认轻柔边框 */
  --border-warm-brown: rgba(74, 62, 61, 0.15); /* 卡片实体边框：暖褐 15% 透明度 */

  /* 核心文字（深暖褐代替纯黑，温柔不刺眼） */
  --text: #4A3E3D;                        /* 主文字：深暖褐 */
  --text-secondary: #8A7E7D;              /* 次要文字：中暖褐 */
  --text-tertiary: #B5A9A8;               /* 辅助文字：浅暖褐 */

  /* 马卡龙多色点缀系统 (Chiikawa Palette) */
  --chiikawa-pink: #FFD5E5;               /* 吉伊粉：用于 RLHF、偏好标注、核心高亮 */
  --hachiware-blue: #D2EBF7;              /* 小八蓝：用于数据清洗、多模态、技术指标 */
  --usagi-yellow: #FFF1C5;                /* 乌萨奇黄：用于 Prompt 工程、大模型、核心优势 */
  --momonga-purple: #E8DDF2;              /* 飞鼠紫：用于模型评测、安全合规、教育背景 */

  /* 对应的深色文字（用于 Tag 文本，确保无障碍对比度） */
  --chiikawa-pink-dark: #853D55;
  --hachiware-blue-dark: #2B5E75;
  --usagi-yellow-dark: #7A5F13;
  --momonga-purple-dark: #63437A;
}
```

## 4. 间距系统 (Spacing System)

采用严格的偶数间距系统，确保页面呼吸感与节奏感。

- **Section 垂直间距**：
  - Desktop: `py-32` (128px)
  - Mobile: `py-20` (80px)
- **组件间距 (Gaps)**：
  - 大网格间距：`gap-12` (48px) 或 `gap-8` (32px)
  - 卡片内部间距：`p-8` (32px) 或 `p-10` (40px)
  - 元素微调间距：`mb-4` (16px), `mb-6` (24px)

## 5. 导航规范 (Navigation)

- **样式**：悬浮式玻璃态导航栏 (`navbar-blur`)。
- **背景**：`rgba(250, 248, 245, 0.75)` 结合 `backdrop-blur-md`。
- **边框**：底部 1px 极轻柔暖灰边框 (`border-bottom`)。
- **交互**：
  - 导航项悬停时，文字颜色平滑过渡到深暖褐，下方出现一个可爱的马卡龙色小圆点（通过 Framer Motion 实现平滑滑入）。
  - 移动端采用右侧抽屉式菜单，带有圆润的卡片设计。

## 6. 按钮规范 (Buttons)

所有按钮必须使用超大圆角（胶囊圆角 `rounded-full`），并带有平滑的弹性过渡。

- **主要按钮 (Primary Button)**：
  - **样式**：背景为吉伊粉到飞鼠紫的渐变，文字为深暖褐（`#4A3E3D`），带有 2px 暖褐实体边框。
  - **投影**：`shadow-[3px_3px_0px_0px_rgba(74,62,61,1)]` (日系硬投影)。
  - **交互**：悬停时向上平滑位移 2px，硬投影变长为 `shadow-[5px_5px_0px_0px_rgba(74,62,61,1)]`；点击时向下位移 1px，投影缩短。
- **次要按钮 (Secondary Button)**：
  - **样式**：背景为纯白，文字为深暖褐，带有 2px 暖褐实体边框。
  - **投影**：`shadow-[3px_3px_0px_0px_rgba(74,62,61,0.15)]`。
  - **交互**：悬停时背景变为极淡的马卡龙色（如小八蓝），投影加深。
- **文字/Ghost 按钮**：
  - **样式**：无背景，文字为中暖褐。
  - **交互**：悬停时文字变为深暖褐，并带有微弱的 Q 弹缩放（`scale-105`）。

## 7. 项目卡片规范 (Project Cards)

项目卡片是展示专业能力的核心，必须兼顾“治愈感”与“视觉重量”。

- **整体结构**：采用左右分栏（Desktop） or 上下分栏（Mobile）。
- **左侧图片区域优化**：
  - **背景**：采用马卡龙色块拼接（如小八蓝到飞鼠紫的柔和渐变），并带有手绘风的网格线（Grid Pattern）或虚线装饰。
  - **核心视觉**：中间悬浮一个纯白圆角小卡片（`rounded-2xl`），内嵌精致的 `Lucide` 核心技术图标（如 `Database`），图标周围点缀 2-3 颗黄色手绘风小星星（`Sparkles`）。
  - **视觉重量**：通过“多层卡片叠加 + 实体投影”撑起左侧的视觉重量，彻底告别空洞。
- **右侧文字与截断规则**：
  - **标题**：单行截断 (`truncate`)，悬停时变色。
  - **描述文字**：严格限制为 3 行，使用 Tailwind 的 `line-clamp-3` 规范溢出。
  - **查看详情交互**：在描述文字下方，设计一个精致的“查看详情”小标签，右侧配有可爱的小箭头（`→`），悬停时小箭头会平滑地向右 Q 弹位移。

## 8. 优势卡片规范 (Skills Cards)

解决“白色卡片贴白色背景，对比度不足，层次感弱”的问题。

- **卡片底色**：采用纯白（`#FFFFFF`）。
- **卡片边框**：加上 **2px 的暖褐实体边框** (`border-2 border-text/15`)。
- **实体硬投影 (Solid Shadow)**：
  - 每个卡片根据其主题，配有专属的 **4px 实体马卡龙色硬投影**。
  - 例如：数据清洗卡片配有小八蓝硬投影（`shadow-[4px_4px_0px_0px_var(--hachiware-blue)]`），Prompt 卡片配有乌萨奇黄硬投影（`shadow-[4px_4px_0px_0px_var(--usagi-yellow)]`）。
  - 这种设计让卡片在米白背景上“跃然而出”，层次感极强，且极具日系插画的精致度。
- **交互**：悬停时卡片向上平滑位移 4px，硬投影变长为 8px（如 `shadow-[8px_8px_0px_0px_...]`），营造出可爱的“漂浮感”。

## 9. Tag / Badge 规范 (Multi-color Tags)

拒绝单一颜色 Tag，引入多色 Tag 方案，根据技术栈属性自动匹配：

- **数据/技术类 Tag** (如数据清洗、CLIP、Python)：
  - **配色**：小八蓝背景 (`bg-[#D2EBF7]/60`) + 深蓝文字 (`text-[#2B5E75]`)。
- **模型/对齐类 Tag** (如RLHF、偏好标注、安全合规)：
  - **配色**：吉伊粉背景 (`bg-[#FFD5E5]/60`) + 深粉文字 (`text-[#853D55]`)。
- **Prompt/工程类 Tag** (如Prompt工程、CoT、Few-shot)：
  - **配色**：乌萨奇黄背景 (`bg-[#FFF1C5]/60`) + 深黄文字 (`text-[#7A5F13]`)。
- **评测/分析类 Tag** (如自动化评测、数据分析、EDA)：
  - **配色**：飞鼠紫背景 (`bg-[#E8DDF2]/60`) + 深紫文字 (`text-[#63437A]`)。

## 10. 响应式与无障碍规则 (Responsive & Accessibility)

- **响应式断点**：
  - Mobile (`< 768px`)：单列布局，卡片内边距缩减为 `p-6`，硬投影缩减为 3px，字体大小按比例缩小。
  - Tablet (`768px - 1024px`)：双列布局，间距微调。
  - Desktop (`> 1024px`)：完整多列布局，最大宽度限制为 `max-w-6xl` (1152px)，确保在大屏上不会过度拉伸。
- **对比度无障碍**：
  - 所有马卡龙色背景上的文字，必须使用对应的深色文字 Token（如吉伊粉背景配深粉文字），确保 HR 和面试官在任何屏幕下都能清晰阅读。

## 11. 动效原则 (Motion Principles)

- **缓动曲线**：全站统一使用治愈系 Q 弹缓动曲线 `cubic-bezier(0.34, 1.56, 0.64, 1)`，营造可爱的弹性。
- **漂浮感 (Floating)**：背景光斑与小装饰元素（星星、大脑、数据库）带有不同速度的缓慢上下漂浮动画。
- **复制成功 Toast**：点击复制邮箱时，在点击位置上方弹出一个带 Q 弹动画的治愈系小气泡 `(•͈⌔•͈⑅) 邮箱已复制啦！`，并在 2 秒后缩放消失。
